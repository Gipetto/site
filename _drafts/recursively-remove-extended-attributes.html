Every once in a while I run in to a problem with downloaded files having extended attributes of com.apple.quarantine that prohibit the files from being opened. com.apple.quarantine is a flag that OS X uses to label applications that have been downloaded from the internet so that it can throw a dialog asking you to confirm opening the application, but this also has the side effect of disallowing the file from being accessed by a web browser or included in to a PHP application.

For 1 or 2 files the files can be opened to trigger the dialog, but for larger libraries this becomes really troublesome.

This is a bash snipped that will recursively remove all extended attributes from a directory. Add it to your .profile or .bashrc and run it from the directory where you're having issues.

rmextattr() {
find . | while read filename
do
while read attr
do
echo "removing ${attr} from ${filename}"
xattr -d "${attr}" "${filename}"
done < <(xattr "${filename}")
done
}
